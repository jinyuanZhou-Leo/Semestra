Semestra is a full-stack scheduling and academic planner with a Python backend and modern web frontend.
The repository root now exposes convenience npm entrypoints for starting the frontend and backend, while frontend package management still lives under `frontend/`.
`backend/` provides FastAPI APIs, data models, persistence, schema-level widget layout validation, env-backed JWT signing plus HttpOnly auth-cookie session transport, Program-level subject-code color-map persistence with stable assignment locking for existing codes plus collision-aware picks for newly discovered codes, account-wide course-resource file storage with local-disk persistence, saved external URL resources, and a 50MB quota, alongside an Alembic-managed schema migration path instead of runtime DDL rewrite helpers. The LMS layer now supports multiple encrypted provider-neutral integrations per user, Program-level integration selection, Course-to-LMS linking through dedicated metadata rows, LMS course import into Programs or Semesters, and read-only Navigation, Announcement, Module, Assignment, Page, Quiz, Syllabus, and Calendar aggregation APIs with Canvas as the first adapter, while provider-specific config normalization and credential masking now live inside the provider adapter layer instead of shared schemas/services. Course CRUD now also validates semester reassignment against the owning Program and correctly recomputes stats for Program-level courses even when `semester_id` becomes null. Account backup import/export is now split so `main.py` keeps thin route wrappers while a dedicated backend transfer module owns the full restore/export pipeline for LMS/resources/schedule/todo data, and the API layer is further modularized across focused auth, schedule, layout, and shared-helper modules so no single backend entry file exceeds 1k lines. The production deploy script now loads the same systemd env file before migrations so schema upgrades target the live database and legacy SQLite deployments receive explicit backfill revisions for missing `programs.subject_color_map`, `courses.color`, `course_resource_files`, and LMS schema changes. The frontend now also includes a Canvas-only course navigation plugin that mirrors the Canvas web app with a sticky course-menu rail, a Home view defined by Canvas `default_view`, hidden assignments/grades/files/discussion tab ids, native Announcements/Modules/Pages/Quizzes/Syllabus views, all-open module sections with memoized rows, instant body swaps instead of height-collapse animation, and offscreen-friendly intrinsic sizing for long lists, shadcn alert treatment for locked pages, host-aligned unavailable empty-state sizing, CTA-only handling for unknown internal tabs plus external tools, same-course page links kept inside Semestra, Canvas-specific page-link metadata preserved in sanitized HTML, richer Canvas table/image support with preserved inline styles, a tuned typography layer for Canvas rich-text bodies, a host-aware sticky rail offset that clears the Course Homepage title bar, LMS-origin resolution for relative Canvas links, and a longer-lived client cache so revisiting pages does not hit the API every time.
The frontend also layers business-level semantic empty-state wrappers on top of shadcn primitives so create, no-results, not-found, and unavailable states share one design language without modifying `components/ui/`, and standardizes ordinary settings forms around shadcn `Field` patterns with descriptions/errors grouped consistently and switch controls aligned to the right. The settings workspace now exposes reusable LMS integration cards, with Canvas still the only LMS UI while provider-specific payload mapping lives in a settings-local provider-definition layer instead of the generic manager component. Program settings now live on a dedicated breadcrumb-aware route with separate LMS and general sections while still selecting and locking the active LMS per Program, Program Dashboard now surfaces that settings navigation as a right-aligned shadcn-style action, includes unassigned Program-level courses in its All Courses list and credit summary, adds a pencil-triggered course edit mode with per-row destructive deletion confirmed through shadcn `AlertDialog`, and GPA Percentage display is now normalized through a shared host-agnostic formatter so pages, dialogs, backup preview, and plugins all keep one decimal place without coupling plugin code to route-level implementations. Course settings manage link/sync/unlink actions, Add Course/Add Semester now use height-stable tabbed import flows with a shared LMS picker that supports year filtering, shadcn-native search/select chrome, a single input-radius results shell, unclipped focus treatment, and visible disabled states for LMS courses already linked inside the Program, Add Course can additionally parse ICS files into Program or Semester courses and now asks for confirmation before manually creating a duplicate course name, Add Semester separates empty/calendar/LMS creation paths, shared settings CRUD panels now keep horizontal scrolling contained inside their cards with shrink-safe section columns, stacked mobile headers, and tighter mobile minimum widths for wider tables, and the Course Resources plugin now keeps its add-resource dialog height stable across upload/link tabs while deleting saved-link resources without tripping backend storage-root unlink errors. The Calendar tab now loads schedule and LMS overlays through buffered date-range queries instead of whole-semester fan-out, keeps Calendar settings from eagerly preloading every semester week just to seed export UI, titles LMS events with local course codes, lets users enable/disable each registered Calendar source from a unified source list with integrated per-source colors and source-specific settings sections, revalidates source data when a previously hidden source is turned back on, treats Programs without LMS configuration as an empty external feed instead of a failing request, opens LMS/read-only events in a shared detail dialog, exposes a destructive confirmed setting for rich LMS description rendering, defaults LMS descriptions to safe text/list rendering unless that setting is enabled, preserves common block structure in that safe mode, safely renders richer HTML with limited inline-style preservation inside a fixed-height scroller, and removes the extra outer wrapper around event metadata, while the Gradebook assessment editor now supports either direct percentages or earned/possible points, keeps the manual panel scrollable while the LMS panel stays fixed-height with only the assignment list scrolling, replaces nested score tabs with a compact radio-group control, blocks all gradebook math unless total assessment weight is exactly 100%, can still add LMS assignments as one-time local rows using provider-normalized LMS due dates instead of syncing them, and backup restore preview now reflects the expanded backup format across LMS integrations, Program-level courses, todo data, resources, schedule-bearing course content, and point-based gradebook inputs.
`frontend/` provides the user interface, including RGL v2 container-width-based widget sizing, lower-frequency resize-guarded relayouts, zero-width mount protection on dashboard remount,
