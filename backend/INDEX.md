Backend exposes FastAPI endpoints and coordinates auth, CRUD, and domain logic.
Data contracts are defined in Pydantic schemas and persisted through SQLAlchemy models, with Alembic now owning schema evolution instead of startup-time compatibility rewrites.
Auth now signs JWTs from env-backed secrets and ships browser sessions via HttpOnly cookies while still validating bearer tokens server-side.
Program-level subject-code color maps now persist in the backend as stable locked assignments so Course List, Program Dashboard, Course Settings, Todo, and backups share the same automatic/default course colors while new subject codes avoid collisions when palette room remains.
Course resources now store file metadata in SQLite, file bytes on local disk, saved external URLs as link-only records, and enforce a 50MB account-wide quota across every course resource upload.
LMS integrations now persist multiple encrypted provider-neutral connections per user, let each Program choose one active integration, and attach Courses to LMS data through dedicated link rows rather than overloading the local Course entity.
The backend now exposes generic LMS course import, Course link/relink/unlink, ICS-backed semester/course imports, and read-only Assignment plus Calendar APIs without leaking secrets; semester LMS calendar reads now degrade to an empty list when a Program has no LMS configured, while Calendar-facing semester schedule reads now use a dedicated date-range endpoint instead of one-request-per-week fan-out, with Canvas implemented through a REST adapter that owns provider-specific config normalization and credential masking before data reaches shared LMS consumers.
Account backup import/export now restores current persisted features end to end, including LMS integrations plus course links, Program-level unassigned courses, semester date windows and Reading Week, course schedule structures, semester todo state, course resources with file bytes, gradebook LMS provenance, and background plugin preload preferences, while the heavy backup serialization/restore flow now lives in a dedicated transfer module instead of bloating the FastAPI entrypoint.
The backend API layer is now split across focused router/support modules so `main.py` stays below 1k lines: auth or user or LMS or backup routes, schedule or event routes, widget or tab routes, shared API validators, and schedule helper logic each live in dedicated files, while the slimmer entrypoint now explicitly imports `datetime`, `timedelta`, and `Response` where the remaining semester import and resource routes still use them.
Course-scoped widget/tab ownership checks now resolve through `Course -> Program` directly so Program-level courses without a bound semester still support Course homepage initialization and tab/widget CRUD, while course updates now validate semester reassignment against the owning Program and recalculate stats correctly when a course is left unassigned.
