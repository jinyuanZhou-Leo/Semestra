Application source root containing entrypoints, routing shell, and domain modules.
Feature behavior is composed from components, contexts, hooks, services, and utilities with TanStack Query-backed server-state caching, context-safe optimistic synchronization, and business-layer semantic empty-state wrappers.
Auth boot now relies on credentialed cookie sessions instead of browser-stored bearer tokens, with central session refresh and sign-out handling.
The standalone `calendar-core/` domain layer now sits beside `plugins/` so Calendar event contributions can register through a dedicated API rather than the plugin loader.
The plugin-system now keeps host-coupled built-in behavior out of the public plugin contract by separating plugin-global settings from tab/widget instance settings, while semester/course settings pages now resolve missing tab-plugin settings to terminal unavailable states instead of endless loading placeholders.
Plugin-global settings can now opt into framework-managed shared JSON persistence with the same debounce/max-wait sync semantics used by tab and widget instance settings, while builtin plugins can still keep host-coupled custom save flows when needed; shared autosave now pauses after a failed attempt until the user changes the draft again so invalid or broken requests do not loop indefinitely.
Authenticated sessions can now optionally preload still-idle plugins in the background after the page load event by using browser idle time, with the toggle stored in global user settings.
The settings workspace now also treats LMS integrations as server state by loading saved integration records through TanStack Query, validating drafts before save, and reusing a generic in-place status button for validate/save actions. Canvas remains the only LMS UI, but provider-specific config/credential mapping now lives in a settings-local provider-definition layer instead of `LmsIntegrationManager` itself.
