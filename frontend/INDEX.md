Vite + React frontend for Semestra with lazy routes and plugin-driven dashboards.
Core behavior is split across contexts, hooks, services, reusable UI components, and a standalone `src/calendar-core/` domain layer for Calendar event-source registration.
Build/test configs and scripts support local development, bundling, release metadata, cookie-backed auth bootstrapping, compact calendar/todo loading shells including a simplified height-matched Calendar body placeholder, a single-shell Calendar border treatment with refined scrollbars, centered Apple-style week headers, a lighter Apple-style time axis, borderless Calendar event cards with spaced parallel lanes, dynamic sticky offsets for workspace settings titles, debounced settings auto-save persistence without inline status UI, a TanStack Query-based cache/mutation layer for entity/plugin/todo/gradebook/schedule/LMS server state, buffered range-based Calendar schedule/LMS fetching keyed by the visible window, homepage-aware plugin loading skeletons, opacity-only plugin fade-ins that preserve viewport-fixed dashboard FAB positioning during load, empty-state-safe dashboard width measurement so the first widget appears without refresh, business-layer semantic empty-state wrappers that unify create/no-results/not-found/unavailable feedback without modifying shadcn primitives, a Settings-page multi-integration LMS flow that now loads saved integrations through TanStack Query, validates drafts before save, pauses shared autosave retries after failures until the draft changes again, and reuses a generic in-place async status button for validation, Program/Course settings surfaces that manage Program LMS selection plus Course link/sync/unlink actions, Program Dashboard now exposes a right-aligned shadcn-style settings action, includes unassigned Program-level courses in All Courses summaries, adds a pencil-triggered edit mode with destructive per-row course deletion confirmed through shadcn `AlertDialog`, and Add Course/Add Semester flows now use height-stable tabs with a shared LMS search-plus-year-filter picker, shadcn-native search/select chrome, a single input-radius results shell, unclipped focus rings, overflow-safe rows, and a three-path empty/calendar/LMS semester creation split, plus an ICS-backed course import path inside the course manager dialog and duplicate-name confirmation before manual course creation, an LMS-backed read-only Calendar source, an Apple Reminder-style Todo aggregate with section-scoped compact composers plus shared semester query hydration, canonical-cache-backed Calendar-to-Todo sync signals that no longer publish UI-shaped todo storage across module boundaries, shared row-shell typography between inline create and task rows, per-composer local inline-create drafts, in-place title editing with blur-to-save task creation when a draft already has a title, compact time-chip editing that stays left-aligned while open, completed-summary controls, swipe-safe deletion, locally persisted per-list sort preferences, no self-refresh waterfall after successful todo mutations, and mirrored semester/course persistence, stable persisted Program subject-color assignments that keep old course-code colors fixed while giving new codes the next free palette color when possible, a Gradebook command-center refresh with fact-only gradebook payloads plus client-derived projections, shared query caching across tab/settings surfaces, explicit total-weight gating that turns Gradebook top stats red and suppresses calculations below 100%, homepage tab orchestration that now keeps Dashboard first and Settings last while allowing Gradebook, Calendar, Schedule, and Todo to be reordered, selective `user-select: none` protection on Gradebook stat/control surfaces without affecting editable inputs, a Course List-style percentage-score assessment shell, and the split habit-streak dual-widget runtime with isolated per-instance streak data so the browser no longer persists login bearer tokens in localStorage.
